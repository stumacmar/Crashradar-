What this implementation actually does

  • Pulls history for each indicator from FRED (where available) using your API key. [oai_citation:0‚Ä°FRED](https://fred.stlouisfed.org/docs/api/fred/?utm_source=chatgpt.com)
  • Computes stress-aligned z-scores: each series standardized on its own history since 1985 (configurable), oriented so higher = more stress.
  • Within each bucket (Leading / Financial / Nowcast), indicators are equal-weighted.
  • Composite raw index: ESI_raw = 0.6¬∑Z_leading + 0.3¬∑Z_financial + 0.1¬∑Z_nowcast.
  • ESI_z = (ESI_raw ‚àí Œº_ESI) / œÉ_ESI, where Œº_ESI and œÉ_ESI must be estimated from historical ESI_raw. In this file they default to 0/1 until you run a real calibration.
  • No ‚ÄúFed-approved‚Äù marketing. Thresholds and mappings are documented and auditable.
